# Roadmap

## Product Direction
Cheery Cubs is moving from a hobby app toward a real launch product.

The roadmap should stay focused on:

- clear parent trust
- meaningful age and skill guidance
- distinct game mechanics
- quality over raw game count

Core catalog model:

- `6` top-level categories
- `10` games per category
- `60` games total

Important rule:

- the target is `10 distinct games` in each category
- this does not mean `10 quiz reskins` or `10 content swaps`
- each category should feel broad, credible, and intentionally designed

## Phase 1: Launch Foundation
Goal: make the app feel structured, real, and ready for serious testing.

Focus:

- parent-first discovery
- stage-based browsing
- starter baby co-play section
- strong core catalog
- stable deploy/build flow
- first documentation set

Key outputs:

- home and browse flow
- stage metadata
- skill metadata
- core launch games
- docs and handoff notes

## Phase 2: Category Strength

## Phase 3: 60-Game Core Catalog
Goal: reach a balanced 60-game catalog across 6 top-level categories.

Target:

- `10` games per category
- `60` total games

Categories:

- `Baby Co-Play`
- `Letters & Reading`
- `Numbers & Counting`
- `Thinking & Patterns`
- `Memory & Focus`
- `World Around Me`

Important rule:

- do not reach 60 by padding the catalog with near-duplicates
- each game should feel mechanically or developmentally distinct

Success means:

- each category can stand on its own as a believable mini-library
- a parent can see variety inside a category, not repetition
- the catalog feels curated, not inflated

## Beyond Phase 3
Once the 60-game core is strong, possible next product tracks:

- dashboard insights by skill
- guided learning paths
- premium subscription model
- more polished visual asset system
- app store onboarding and screenshots
- curriculum-based recommendation engine
